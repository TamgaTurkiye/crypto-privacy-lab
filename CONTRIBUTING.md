# Contributing

Crypto Privacy Lab'e katkılar GitHub Issues ve Pull Requests üzerinden alınır.

Genel Tamga katkı kuralları için organizasyon rehberine bakın:
https://github.com/TamgaTurkiye/.github/blob/main/CONTRIBUTING.md

## Contribution Flow

1. Açık bir issue seçin.
2. Issue altında çalışmak istediğinizi belirtin.
3. Repo’yu fork edin.
4. Kendi branch’inizde değişiklik yapın.
5. Pull Request açın.
6. Maintainer review sonucuna göre düzeltme yapın.

## Repo-Specific Rules

- Gerçek key, token, password, credential veya leaked data kullanmayın.
- Hash cracking, password list, credential abuse veya breaking workflow yazmayın.
- “Kendi kriptonu yaz” önerisinde bulunmayın.
- Privacy-by-design içeriklerini hukuki tavsiye gibi yazmayın.
- Post-quantum içeriklerinde panik dili kullanmayın.
- Her teknik öneri kaynakla desteklenmelidir.

## Trust Tiers

- Tier 1: Kavramsal açıklama ve checklist.
- Tier 2: Secure storage, key management, TLS ve privacy design notları.
- Tier 3: Crypto-sensitive, post-quantum veya failure case konuları; maintainer review zorunludur.

## Contributor Rights

Her PR, `NOTICE.md` ve organizasyon seviyesindeki contributor rights ilkelerini kabul etmiş sayılır.
